**사용자 제공 구조:**
```
행 1-15: 요약 대시보드
행 16: 열 헤더 (A:작성일시, B:구역, ..., H:민원내용, ...)
```

**열 구조:**
```
A(0): 작성일시
B(1): 구역 (동/호수)
C(2): 요청자 유형
D(3): 민원인 신상
E(4): 접수자
F(5): 담당자
G(6): 진행상황 (처리 상태)
H(7): 민원내용 ← 가장 중요!
I(8): 조치사항 (처리 결과)
J(9): 피드백
K(10): 비고
```

**수정 전:**
```javascript
const rows = data.table.rows;
const complaints = rows.map(row => {
  const cells = row.c;
  return {
    date: cells[0]?.v || '-',        // A: 작성일시 ✅ (우연히 맞음)
    location: cells[1]?.v || '-',    // B: 구역 ✅ (우연히 맞음)
    content: cells[2]?.v || '-',     // C: 요청자유형 ❌ (민원내용이 아님!)
    status: cells[3]?.v || '-',      // D: 민원인신상 ❌ (진행상황이 아님!)
    result: cells[4]?.v || '-',      // E: 접수자 ❌ (조치사항이 아님!)
    completeDate: cells[5]?.v || '-' // F: 담당자 ❌ (완료일이 아님!)
  };
});
```

- ✅ 테이블 열 헤더와 데이터 일치:
