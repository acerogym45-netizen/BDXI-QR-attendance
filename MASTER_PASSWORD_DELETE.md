## 🔒 보안 시스템

### 2단계 인증 프로세스

```javascript
app.deleteDocument = async function(documentId) {
  // 1단계: 삭제 의사 확인
  if (!confirm('정말로 이 서류를 삭제하시겠습니까?\n삭제된 서류는 복구할 수 없습니다.')) {
    return; // 취소
  }
  
  // 2단계: 마스터 비밀번호 확인
  if (!this.verifyMasterPassword()) {
    return; // 비밀번호 틀림 또는 취소
  }
  
  // 3단계: 삭제 실행
  await this.sb
    .from('documents')
    .update({ is_active: false })
    .eq('id', documentId);
}
```

### 비밀번호 검증 함수

```javascript
app.verifyMasterPassword = function() {
  const password = prompt('🔐 마스터 관리자 비밀번호를 입력하세요:\n\n※ 문서 삭제는 마스터 권한이 필요합니다');
  
  if (!password) {
    return false; // 취소 버튼
  }
  
  const MASTER_PASSWORD = 'kw1234'; // 마스터 비밀번호
  
  if (password === MASTER_PASSWORD) {
    console.log('✅ 마스터 비밀번호 확인 성공');
    return true;
  } else {
    alert('❌ 비밀번호가 틀렸습니다.\n마스터 관리자만 문서를 삭제할 수 있습니다.');
    return false;
  }
};
```

## 📝 사용 시나리오

### ✅ Scenario 1: 일반 사용자 (비밀번호 모름)

1. 문서 목록에서 "삭제" 버튼 클릭
2. **확인**: "정말로 이 서류를 삭제하시겠습니까?" → [확인]
3. **비밀번호 입력**: 🔐 프롬프트 표시
4. **입력**: `wrong-password` (틀린 비밀번호)
5. **결과**: ❌ "비밀번호가 틀렸습니다" 알림 → 삭제 취소

### ✅ Scenario 2: 마스터 관리자 (비밀번호 알고 있음)

1. 문서 목록에서 "삭제" 버튼 클릭
2. **확인**: "정말로 이 서류를 삭제하시겠습니까?" → [확인]
3. **비밀번호 입력**: 🔐 프롬프트 표시
4. **입력**: `kw1234` (올바른 비밀번호)
5. **결과**: ✅ "서류가 삭제되었습니다" 알림 → 삭제 완료

### ✅ Scenario 3: 취소

1. 문서 목록에서 "삭제" 버튼 클릭
2. **확인**: "정말로 이 서류를 삭제하시겠습니까?" → [취소]
3. **결과**: 아무 일도 일어나지 않음

또는

1. 삭제 확인 → [확인]
2. **비밀번호 입력**: 🔐 프롬프트 표시 → [취소]
3. **결과**: 삭제 취소

## 🎯 장점

### 1. 보안성 ✅
- 비밀번호 없이는 삭제 불가능
- 2단계 확인으로 실수 방지

### 2. 사용성 ✅
- 모든 사용자가 삭제 버튼을 볼 수 있음
- 권한이 있는 사용자는 즉시 삭제 가능
- 권한이 없는 사용자는 비밀번호 입력으로 차단

### 3. 투명성 ✅
- 삭제 기능이 숨겨져 있지 않음
- 명확한 피드백 (비밀번호 틀림 알림)

### 4. 복구 가능성 ✅
- Soft Delete 방식 (`is_active = false`)
- 필요 시 DB에서 복구 가능
