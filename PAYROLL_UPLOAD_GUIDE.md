## 시스템 개요

### 기능 요약
- **관리자**: 급여명세서 PDF 업로드 → 직원에게 배포
- **직원**: employee-app.html에서 급여명세서 조회/다운로드
- **추적**: 조회수, 다운로드수, 상태 관리

### 시스템 흐름
```
[관리자 index.html]
    ↓ PDF 업로드
[Supabase Storage: payroll-statements]
    ↓ URL 저장
[payroll_statements 테이블]
    ↓ 직원 접근
[employee-app.html - 급여 탭]
```

## 관리자 기능

### 1️⃣ 급여 관리 탭 접근

**위치**: `index.html` → 상단 네비게이션 → **💰 급여 관리**

**대시보드 통계**:
- 이번 달 발급: 현재 월 발급 건수
- 전체 직원: 활성 직원 수
- 미조회 명세서: `pending` 상태 건수
- 총 발급 건수: 전체 발급 누적

### 2️⃣ 급여명세서 업로드

**버튼**: `급여명세서 업로드` 클릭

**업로드 모달 구성**:

#### 연월 선택
```
- 연도: 현재 -2년 ~ +1년
- 월: 1월 ~ 12월
```

#### 직원 선택 방법 (3가지)

**A. 개별 선택** ⭐ 기본
- 직원 목록에서 체크박스로 선택
- 여러 명 동시 선택 가능
- 선택된 직원 수 실시간 표시

**B. 부서별**
- 드롭다운에서 부서 선택
- 해당 부서 전체 직원에게 발급

**C. 전체**
- 활성화된 모든 직원에게 발급
- 인원 수 미리 확인 가능

#### PDF 파일 업로드
```
- 파일 형식: PDF만 가능
- 최대 크기: 10MB
- 업로드 방법: 클릭 또는 드래그 앤 드롭
```

**⚠️ 중요**: 
- **동일한 파일**이 선택된 모든 직원에게 발급됩니다
- 직원별 개별 파일 업로드는 각각 별도로 진행해야 합니다

### 3️⃣ 업로드 실행

**프로세스**:
1. 연월, 직원, 파일 선택 완료
2. "업로드 시작" 버튼 클릭
3. 확인 메시지: "X명의 직원에게 발급하시겠습니까?"
4. 진행률 표시: 각 직원별 처리 상태
5. 완료 메시지: 성공/실패 건수

**진행 중**:
```
김철수 처리 중... 45%
```

**완료**:
```
✅ 업로드 완료!

성공: 15건
실패: 0건
```

### 4️⃣ 급여명세서 관리

**목록 화면**:
```
┌─────────────────────────────────┐
│ 2026년 5월                      │ 📋 대기
│ 김철수 (대리)                    │
│ 총무부                          │
├─────────────────────────────────┤
│ 조회: 0회 | 다운로드: 0회 | 5/11 │
│ 최근 조회: 조회 없음              │
├─────────────────────────────────┤
│ [조회] [🗑️]                     │
└─────────────────────────────────┘
```

**기능**:
- **조회**: PDF 새 탭에서 열기
- **삭제**: 명세서 영구 삭제 (확인 필요)
- **새로고침**: 목록 다시 로드
- **엑셀 내보내기**: 전체 현황 엑셀 다운로드

### 5️⃣ 필터 기능

**필터 옵션**:
- 검색: 직원 이름, 부서
- 연도: 전체 / 특정 연도
- 월: 전체 / 1~12월
- 부서: 전체 / 특정 부서
- 상태: 전체 / 대기 / 조회 / 다운로드

**예시**:
```
검색: "김"
연도: 2026년
월: 5월
부서: 총무부
상태: 대기

→ 2026년 5월 총무부 김씨 성을 가진 직원 중 미조회 명세서만 표시
```

### 6️⃣ 엑셀 내보내기

**내보내기 데이터**:
- 연월
- 직원명, 사번, 부서, 직급
- 상태, 조회수, 다운로드수
- 최근 조회 시간, 발급일

**파일명**: `급여명세서_현황_2026-05-11.xlsx`

## 주의사항

### ⚠️ 중요 제약사항

#### 1. 동일 PDF 파일 배포
- **현재 버전**: 하나의 PDF가 모든 선택된 직원에게 동일하게 발급됨
- **개별 명세서 필요 시**: 각 직원마다 별도로 업로드 필요

**예시**:
```
❌ 잘못된 사용:
- 김철수, 이영희, 박민수를 선택하고
- 김철수 명세서.pdf를 업로드
→ 3명 모두 김철수 명세서를 받게 됨!

✅ 올바른 사용:
1. 김철수 선택 → 김철수.pdf 업로드
2. 이영희 선택 → 이영희.pdf 업로드
3. 박민수 선택 → 박민수.pdf 업로드
```

#### 2. 파일명 규칙
Supabase Storage에 저장되는 파일명:
```
{연월}_{사번}_{이름}.pdf

예: 2026-05_EMP001_김철수.pdf
```

- 동일 연월, 동일 직원 재업로드 시: 기존 파일 **덮어쓰기** (upsert: true)
- 중복 방지를 위해 사번 또는 ID 사용

#### 3. Storage 버킷 필수
**사전 준비**:
```
Supabase Dashboard → Storage → Create Bucket

버킷명: payroll-statements
Public: No (Private)
File size limit: 10MB
Allowed MIME types: application/pdf
```

**RLS 정책** (자동 생성 권장):
```sql
-- 관리자만 업로드 가능
CREATE POLICY "Admins can upload"
ON storage.objects FOR INSERT
TO authenticated
WITH CHECK (bucket_id = 'payroll-statements');

-- 직원은 자신의 파일만 읽기
CREATE POLICY "Employees can read own files"
ON storage.objects FOR SELECT
TO authenticated
USING (bucket_id = 'payroll-statements');
```

#### 4. 데이터베이스 테이블 필수
```sql
-- 실행 필요: database/CREATE_PAYROLL_SYSTEM.sql
```

테이블 구조:
- `payroll_statements`: 급여명세서 메타데이터
- `employee_login_logs`: 직원 로그인 기록
- `payroll_notifications`: 알림 로그 (선택)

## FAQ

### Q1. 업로드 후 직원이 볼 수 없다고 합니다

**확인사항**:
1. **Storage 버킷 생성 확인**
   - Supabase Dashboard → Storage
   - `payroll-statements` 버킷 존재 여부

2. **RLS 정책 확인**
   - `payroll_statements` 테이블 RLS 활성화
   - 직원 조회 정책 존재 여부

3. **employee-app.html 로그인 확인**
   - 직원이 제대로 로그인했는지
   - `employee_id`가 올바르게 저장되었는지

4. **연월 확인**
   - 업로드한 연월과 직원이 보는 필터 연월 일치 여부

### Q2. 업로드 중 실패가 발생합니다

**원인 및 해결**:

**A. "Authentication failed"**
```
원인: Supabase Storage 권한 부족
해결: RLS 정책 확인 및 추가
```

**B. "File too large"**
```
원인: 파일 크기 > 10MB
해결: PDF 압축 또는 Storage 설정 변경
```

**C. "Employee not found"**
```
원인: 직원 ID가 잘못됨
해결: employees 테이블 데이터 확인
```

**D. "Bucket not found"**
```
원인: payroll-statements 버킷 미생성
해결: Supabase Storage에서 버킷 생성
```

### Q3. 직원별 개별 PDF를 어떻게 업로드하나요?

**방법 1: 개별 선택 반복** (추천)
```
1. 김철수 선택 → 김철수.pdf → 업로드
2. 이영희 선택 → 이영희.pdf → 업로드
3. 박민수 선택 → 박민수.pdf → 업로드
...
```

**방법 2: 향후 개선 (구현 예정)**
- ZIP 파일 업로드 → 자동 매칭
- 엑셀 + PDF 일괄 업로드

### Q4. 잘못 업로드한 명세서를 삭제하려면?

**단계**:
1. 💰 급여 관리 탭 이동
2. 목록에서 해당 명세서 찾기 (필터 활용)
3. 🗑️ 삭제 버튼 클릭
4. 확인 메시지: "정말 삭제하시겠습니까?" → 확인

**⚠️ 주의**:
- 삭제는 **되돌릴 수 없습니다**
- Storage 파일과 DB 레코드 모두 삭제됨
- 직원이 이미 다운로드한 파일은 직원 로컬에 남아있음

### Q5. 조회수/다운로드수가 증가하지 않습니다

**확인사항**:
1. **직원이 실제 조회/다운로드했는지**
   - employee-app.html 급여 탭 접근
   - 조회 또는 다운로드 버튼 클릭

2. **JavaScript 함수 확인**
   - `viewPayroll()`: last_viewed_at 업데이트
   - `downloadPayroll()`: download_count 증가

3. **브라우저 캐시**
   - 관리자 페이지 새로고침 (F5)
   - "새로고침" 버튼 클릭

### Q6. 알림 기능은 언제 구현되나요?

**현재 상태**: 체크박스만 있고 기능은 미구현

**향후 구현 예정**:
- 이메일 알림 (SendGrid/AWS SES)
- SMS 알림 (Twilio)
- 푸시 알림 (PWA)

**구현 시 동작**:
```
급여명세서 업로드 완료
    ↓
직원에게 자동 알림 발송
    ↓
"2026년 5월 급여명세서가 발급되었습니다"
```

### Q7. 통계가 정확하지 않습니다

**통계 업데이트 시점**:
- 탭 전환 시: `switchTab('payroll')` → `loadPayrollStatistics()`
- 새로고침 버튼 클릭 시
- 업로드 완료 후

**수동 새로고침**:
```
💰 급여 관리 탭 → 새로고침 버튼 클릭
```

### Q8. 엑셀 내보내기가 안 됩니다

**필요 라이브러리**: SheetJS (xlsx.full.min.js)

**확인**:
```html
<script src="https://cdn.sheetjs.com/xlsx-0.20.1/package/dist/xlsx.full.min.js"></script>
```

**브라우저 호환성**:
- Chrome, Edge, Firefox: ✅ 지원
- Safari: ✅ 지원
- IE11: ❌ 미지원
