# 🎯 Before/After 자동 인식 가이드

ESP32-CAM v3.0 펌웨어 - Before/After 자동 쌍 업로드 기능

---

## 🔥 새로운 기능

### v3.0의 핵심 기능

```
✅ 버튼 1회: Before 사진 촬영 및 임시 저장
✅ 버튼 2회: After 사진 촬영 + Before/After 자동 쌍 업로드
✅ 자동으로 같은 작업의 Before/After 매칭
✅ 관리자 페이지에서 쌍으로 표시
✅ 작업 시간 자동 계산
```

---

## 🎮 사용 방법

### 작업자 워크플로우

```
1️⃣ 작업 전 (Before)
   ↓
   더러운 필터 앞에서
   ↓
   손목 버튼 한 번 "딸깍"
   ↓
   LED 1회 깜빡임 + 짧은 진동
   ↓
   화면: "Before 저장 ✓"
   ↓
   ⏱️ 1초 소요

2️⃣ 작업 진행
   ↓
   양손 자유롭게 필터 청소
   ↓
   (ESP32-CAM은 After 대기 중)

3️⃣ 작업 후 (After)
   ↓
   깨끗한 필터 앞에서
   ↓
   손목 버튼 다시 한 번 "딸깍"
   ↓
   LED 5회 빠른 깜빡임 + 긴 진동
   ↓
   화면: "After 저장 ✓"
   ↓
   자동으로 Before/After 쌍 업로드!
   ↓
   ⏱️ 1초 소요 (총 2초)

4️⃣ 관리자 페이지 확인
   ↓
   Before/After 사진이 나란히 표시됨
   ↓
   작업 시간도 자동 계산됨
```

## 💡 LED 피드백 신호

### 상태별 LED 패턴

| 상태 | LED 패턴 | 진동 | 의미 |
|------|----------|------|------|
| Before 촬영 완료 | ● (1회 깜빡임, 0.5초) | 짧게 (0.1초) | Before 저장됨, After 대기 |
| After 촬영 완료 | ●●●●● (5회 빠른 깜빡임) | 길게 (0.5초) | 쌍 완료, 업로드 성공 |
| 에러 | ●●●●● (5회 느린 깜빡임) | 없음 | 업로드 실패 |
| Before 타임아웃 | ●●● (3회 중간 속도) | 없음 | 5분 경과, 초기화됨 |

## 🔄 상태 머신

### 시스템 상태 흐름

```
┌─────────────────┐
│  STATE_READY    │ ← 초기 상태
│  (대기 중)       │
└────────┬────────┘
         │
    버튼 1회 누름
         │
         ↓
┌─────────────────┐
│STATE_BEFORE_    │
│  TAKEN          │ ← Before 촬영 완료
│(After 대기 중)  │
└────────┬────────┘
         │
    버튼 1회 더 누름
         │
         ↓
┌─────────────────┐
│STATE_PAIR_      │
│  COMPLETE       │ ← Before/After 쌍 완료
│(자동 업로드)    │
└────────┬────────┘
         │
   자동으로 2초 후
         │
         ↓
┌─────────────────┐
│  STATE_READY    │ ← 다음 작업 준비
│  (대기 중)       │
└─────────────────┘
```

## ⚠️ 타임아웃 기능

### 5분 자동 초기화

```
Before 촬영 후 5분이 지나면:
↓
자동으로 상태 초기화
↓
LED 3회 중간 속도 깜빡임
↓
다시 Before부터 시작해야 함

이유:
- 작업을 잊어버린 경우
- 작업이 취소된 경우
- Before만 찍고 After를 안 찍은 경우
```

## 🔧 v2.0 → v3.0 업그레이드

### 주요 차이점

| 항목 | v2.0 (기본) | v3.0 (Before/After) |
|------|-------------|---------------------|
| 촬영 방식 | 버튼 누를 때마다 독립 사진 | 2회 눌러서 쌍 생성 |
| 업로드 | 즉시 개별 업로드 | Before 임시 저장 → After와 함께 쌍 업로드 |
| 메타데이터 | photo_url (단일) | photo_urls (배열) |
| 관리자 화면 | 개별 사진 나열 | Before/After 나란히 표시 |
| 작업 시간 | 기록 안됨 | 자동 계산 및 저장 |

### 업그레이드 방법

```
1. v3.0 펌웨어 다운로드
   esp32_camera_firmware_v3_before_after.ino

2. Arduino IDE에서 열기

3. WiFi/작업자 정보 수정 (v2와 동일)

4. ESP32-CAM에 업로드

5. 완료! (하드웨어 변경 없음)
```

## 🎓 작업자 교육

### 30초 교육 스크립트

```
강사: "이제 Before/After를 자동으로 찍어줍니다"

[실물 시연]
1. "작업 전에 버튼 한 번" (딸깍)
   → LED 1회 깜빡임

2. "작업하세요" (손은 자유롭게)

3. "작업 끝나면 버튼 한 번 더" (딸깍)
   → LED 5회 깜빡임

4. "끝! 폰에서 Before/After가 자동으로 나타납니다"

작업자: "오! 쉽네요!"
```

### 주의사항

```
⚠️ Before 찍고 5분 안에 After를 찍어야 함
   → 안 그러면 자동 초기화됨

⚠️ Before 찍은 후 다른 위치 가지 말 것
   → 같은 위치의 Before/After여야 함

✅ 여러 위치 작업 시:
   위치1 Before → After 완료
   ↓
   위치2로 이동
   ↓
   위치2 Before → After 완료
   ↓
   (반복)
```

## 🐛 문제 해결

### Q1: Before 찍었는데 LED가 계속 깜빡여요

```
A: Before가 저장됐다는 신호입니다.
   이제 작업하고 After를 찍으면 됩니다.
```

### Q2: Before 찍고 5분 지났는데 LED가 3번 깜빡였어요

```
A: 타임아웃되어 초기화됐습니다.
   다시 Before부터 시작하세요.
```

### Q3: After 찍었는데 업로드 안돼요 (LED 5회 느리게)

```
A: WiFi 연결 문제입니다.
   1. WiFi 신호 확인
   2. 공유기와 거리 확인
   3. ESP32-CAM 재부팅
```

### Q4: 관리자 페이지에서 Before/After가 따로 보여요

```
A: v2.0 펌웨어를 사용 중입니다.
   v3.0으로 업그레이드하면 자동으로 쌍으로 표시됩니다.
```

## 📊 성능 비교

### v2.0 vs v3.0

| 항목 | v2.0 | v3.0 |
|------|------|------|
| 촬영 속도 | 3초 (각 사진) | 2초 (쌍) |
| 작업 중단 | 2회 (Before, After 각각) | 2회 (동일) |
| 업로드 횟수 | 2회 (각각) | 1회 (쌍으로) |
| 네트워크 사용 | 높음 | 낮음 (50% 절약) |
| 관리 편의성 | 낮음 (수동 매칭) | 높음 (자동 매칭) |

## 🚀 향후 업그레이드 계획

### v4.0 예정 기능

```

  → Before/After가 같은 위치인지 자동 검증

  → 3장 이상의 진행 상황 촬영
```
