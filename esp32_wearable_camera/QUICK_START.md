## 💻 Step 3: 펌웨어 업로드

### Arduino IDE 설정

```
1. 파일 → 환경설정
   → 추가 보드 매니저 URLs 추가:
   https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json

2. 도구 → 보드 → 보드 매니저
   → "esp32" 검색 → ESP32 설치

3. 도구 → 보드 선택:
   AI Thinker ESP32-CAM

4. 도구 → Upload Speed: 115200
5. 도구 → Flash Frequency: 80MHz
6. 도구 → Partition Scheme: Huge APP (3MB No OTA)
7. 도구 → 포트: (FTDI가 연결된 COM 포트 선택)
```

### 코드 수정 (필수!)

`esp32_camera_firmware_v2.ino` 파일에서:

```cpp
// WiFi 설정
const char* WIFI_SSID = "작업장_WiFi_이름";      // ⚠️ 여기 수정
const char* WIFI_PASSWORD = "WiFi_비밀번호";     // ⚠️ 여기 수정

// 작업자 정보
const char* EMPLOYEE_NAME = "홍길동";            // ⚠️ 작업자 이름
const int APARTMENT_ID = 1;                      // ⚠️ 시설 ID
```

### 업로드 절차

```
1. IO0 핀을 GND에 연결 (점퍼 와이어로)
2. ESP32-CAM 리셋 버튼 누름 (작은 버튼)
3. Arduino IDE에서 "업로드" 버튼 클릭
4. "Connecting..." 메시지가 나오면 다시 리셋 버튼 누름
5. "Writing at 0x00001000..." 나오면 업로드 진행 중
6. "Hard resetting via RTS pin..." 나오면 완료
7. IO0-GND 연결 제거
8. 리셋 버튼 한번 더 누름
```

**⚠️ 문제 해결:**
- "Failed to connect": IO0이 GND에 제대로 연결되었는지 확인
- "Timed out waiting for packet header": 리셋 버튼을 다시 누르고 재시도
- "COM port not found": FTDI 드라이버 설치 필요 ([다운로드](https://ftdichip.com/drivers/vcp-drivers/))

## 🚨 문제 해결

### LED 상태 코드

| 패턴 | 의미 |
|------|------|
| 계속 켜짐 | WiFi 연결 완료, 대기 중 |
| 느리게 깜빡임 (2초) | WiFi 연결 실패 |
| 빠르게 깜빡임 (0.5초) | WiFi 연결 시도 중 |
| 5회 빠른 깜빡임 | 촬영/업로드 성공 |
| 10회 빠른 깜빡임 | 에러 발생 |

### 일반적인 문제

#### "카메라 초기화 실패"
```
원인: 카메라 모듈 불량 또는 연결 불량
해결:
1. ESP32-CAM 리셋
2. 카메라 플랫 케이블 재장착
3. 5V 전원 확인 (최소 4.5V 필요)
```

#### "WiFi 연결 안됨"
```
원인: SSID/비밀번호 오류 또는 2.4GHz WiFi 아님
해결:
1. 코드에서 SSID/비밀번호 재확인
2. 2.4GHz WiFi인지 확인 (5GHz 미지원)
3. 공유기와 거리 확인 (최대 20m)
4. 공유기 재부팅
```

#### "업로드 실패 (HTTP 401)"
```
원인: Supabase API 키 오류
해결:
1. Supabase 대시보드에서 API 키 재확인
2. 코드에 정확히 복사
3. Storage bucket 'cleaning-photos' 존재 확인
4. Public 업로드 권한 확인
```

#### "메타데이터 저장 실패"
```
원인: cleaning_tasks 테이블 권한 문제
해결:
1. Supabase 대시보드 → Authentication → Policies
2. cleaning_tasks 테이블에 INSERT 권한 추가
3. 또는 RLS (Row Level Security) 비활성화 (테스트용)
```
