## 🔍 테스트 시나리오

### ✅ 1단계: 기본 기능 테스트

#### 1.1 단일 파일 업로드
- [ ] 관리자 페이지 로그인
- [ ] "🛒 구매 요청" 탭 이동
- [ ] 승인된 구매 요청 1개 선택
- [ ] "검수 사진" 섹션에서 "사진 추가" 버튼 클릭
- [ ] 사진 1장 선택 (예: test01.jpg)
- [ ] "업로드" 버튼 클릭
- [ ] **예상 결과**: "✅ 검수 사진 1장이 업로드되었습니다" 메시지 표시
- [ ] **확인**: 모달 닫힘, 사진이 3열 그리드에 표시됨

#### 1.2 다중 파일 업로드 (5장)
- [ ] 동일한 구매 요청에서 "사진 추가" 클릭
- [ ] 사진 5장 동시 선택
- [ ] **예상 결과**: 미리보기에 5장 표시 (1, 2, 3, 4, 5 번호 오버레이)
- [ ] "업로드" 버튼 클릭
- [ ] **예상 결과**: "업로드 중 (0/5)" → "업로드 중 (5/5)" 진행률 표시
- [ ] **예상 결과**: "✅ 검수 사진 5장이 업로드되었습니다"
- [ ] **확인**: 5장 모두 그리드에 표시, 총 6장 (이전 1장 + 새로 5장)

#### 1.3 최대 개수 업로드 (10장)
- [ ] 새로운 구매 요청 선택
- [ ] 사진 10장 선택
- [ ] **예상 결과**: 미리보기 10장 표시
- [ ] "업로드" 클릭
- [ ] **예상 결과**: "✅ 검수 사진 10장이 업로드되었습니다"
- [ ] **확인**: 10장 모두 표시

---

### ✅ 2단계: 파일명 테스트

#### 2.1 한글 파일명
- [ ] 파일명: "검수사진_01.jpg" 선택
- [ ] 업로드
- [ ] **예상 결과**: 성공 (파일명이 자동으로 "1779244068289_0_a7x4k9m2p.jpg" 형식으로 변환됨)
- [ ] **확인**: 브라우저 개발자 도구 → Console → "📁 파일 1 업로드 시작" 로그 확인
- [ ] **확인**: original: "검수사진_01.jpg", safe: "[timestamp]_0_[random].jpg"

#### 2.2 공백 포함 파일명
- [ ] 파일명: "검수 사진 테스트 01.jpg" 선택
- [ ] **예상 결과**: 성공 (안전한 파일명으로 변환)

#### 2.3 특수문자 포함 파일명
- [ ] 파일명: "검수@사진#테스트!.jpg" 선택
- [ ] **예상 결과**: 성공 (안전한 파일명으로 변환)

#### 2.4 매우 긴 파일명
- [ ] 파일명: "이것은_매우_긴_파일명_테스트입니다_123456789_abcdefghijk.jpg" 선택
- [ ] **예상 결과**: 성공 (안전한 파일명으로 변환)

---

### ✅ 3단계: 파일 형식 테스트

#### 3.1 지원 형식 (모두 성공해야 함)
- [ ] JPG 파일 업로드 → 성공
- [ ] JPEG 파일 업로드 → 성공
- [ ] PNG 파일 업로드 → 성공
- [ ] GIF 파일 업로드 → 성공
- [ ] WEBP 파일 업로드 → 성공

#### 3.2 미지원 형식 (모두 에러 메시지 표시해야 함)
- [ ] PDF 파일 선택
- [ ] **예상 결과**: "⚠️ 지원하지 않는 파일 형식입니다: document.pdf\n허용 형식: JPG, PNG, GIF, WEBP"
- [ ] DOCX 파일 선택
- [ ] **예상 결과**: 동일한 에러 메시지
- [ ] TXT 파일 선택
- [ ] **예상 결과**: 동일한 에러 메시지

---

### ✅ 4단계: 파일 크기 테스트

#### 4.1 정상 크기 (모두 성공)
- [ ] 100KB 파일 → 성공
- [ ] 1MB 파일 → 성공
- [ ] 3MB 파일 → 성공
- [ ] 4.9MB 파일 → 성공

#### 4.2 초과 크기 (에러 메시지)
- [ ] 5.1MB 파일 선택
- [ ] **예상 결과**: "⚠️ 파일이 너무 큽니다: large_file.jpg\n최대 5MB까지 가능합니다\n현재 크기: 5.10MB"
- [ ] 10MB 파일 선택
- [ ] **예상 결과**: "현재 크기: 10.00MB" 메시지

### ✅ 5단계: 제한사항 테스트

#### 5.1 파일 미선택
- [ ] 사진 추가 버튼 클릭
- [ ] 파일 선택 없이 "업로드" 클릭
- [ ] **예상 결과**: "📷 사진을 선택해주세요"

#### 5.2 최대 개수 초과
- [ ] 사진 11장 선택
- [ ] **예상 결과**: "⚠️ 최대 10장까지 업로드 가능합니다\n현재 선택: 11장"
- [ ] 사진 15장 선택
- [ ] **예상 결과**: "현재 선택: 15장"

---

### ✅ 6단계: 실제 금액 입력 테스트

#### 6.1 정상 금액 입력
- [ ] 실제 구매 금액 입력: "25000"
- [ ] 사진 1장 선택 및 업로드
- [ ] **예상 결과**: 업로드 성공
- [ ] **확인**: 구매 요청 상세에서 "실제 구매 금액: 25,000원" 표시

#### 6.2 금액 미입력
- [ ] 실제 구매 금액 비워둠
- [ ] 사진 업로드
- [ ] **예상 결과**: 성공 (금액은 업데이트 안 됨)

#### 6.3 잘못된 금액 입력
- [ ] 실제 구매 금액: "-5000" (음수)
- [ ] **예상 결과**: "⚠️ 실제 구매 금액이 올바르지 않습니다"
- [ ] 실제 구매 금액: "abc" (문자)
- [ ] **예상 결과**: 동일한 에러 메시지

---

### ✅ 7단계: 부분 실패 시나리오

#### 7.1 크기 초과 파일 혼합
- [ ] 파일 3장 선택:
  - 파일1.jpg (2MB) ✅
  - 파일2.jpg (6MB) ❌
  - 파일3.jpg (3MB) ✅
- [ ] **예상 결과**: "⚠️ 2/3장이 업로드되었습니다\n\n실패한 파일:\n파일2.jpg: 파일이 너무 큽니다"
- [ ] **확인**: 성공한 2장은 그리드에 표시됨

#### 7.2 형식 오류 파일 혼합
- [ ] 파일 3장 선택:
  - image1.jpg ✅
  - document.pdf ❌
  - image2.png ✅
- [ ] **예상 결과**: "⚠️ 지원하지 않는 파일 형식입니다: document.pdf"
- [ ] **확인**: 업로드 진행 안 됨 (검증 단계에서 중단)

---

### ✅ 8단계: 삭제 기능 테스트

#### 8.1 단일 사진 삭제
- [ ] 업로드된 사진 1장에 마우스 호버
- [ ] **확인**: 우측 상단에 ❌ 버튼 표시
- [ ] ❌ 버튼 클릭
- [ ] **예상 결과**: "이 사진을 삭제하시겠습니까?" 확인 메시지
- [ ] "확인" 클릭
- [ ] **예상 결과**: "✅ 사진이 삭제되었습니다"
- [ ] **확인**: 해당 사진이 그리드에서 사라짐

#### 8.2 삭제 취소
- [ ] 사진 호버 → ❌ 클릭
- [ ] 확인 메시지에서 "취소" 클릭
- [ ] **예상 결과**: 사진 유지됨

#### 8.3 여러 사진 순차 삭제
- [ ] 사진 3장 연속 삭제
- [ ] **확인**: 각 삭제마다 페이지 자동 새로고침
- [ ] **확인**: 삭제된 사진들이 모두 사라짐

---

### ✅ 9단계: UI/UX 테스트

#### 9.1 진행률 표시
- [ ] 사진 5장 업로드 시작

- [ ] **확인**: 각 썸네일에 1, 2, 3 번호 표시
- [ ] **확인**: 썸네일 크기 일정 (w-full h-24)

#### 9.3 모달 동작
- [ ] 업로드 성공 시 모달 자동 닫힘 확인
- [ ] 에러 발생 시 모달 유지 확인 (재시도 가능)

#### 9.4 자동 새로고침
- [ ] 업로드 성공 후
- [ ] **확인**: 상세 페이지 자동 새로고침

### ✅ 10단계: 에지 케이스 테스트

#### 10.1 네트워크 문제 시뮬레이션
- [ ] 개발자 도구 → Network 탭 → Offline 설정
- [ ] 사진 업로드 시도
- [ ] **예상 결과**: "⚠️ 사진 업로드 중 심각한 오류가 발생했습니다"
- [ ] **확인**: 버튼 상태 복구됨

#### 10.2 페이지 새로고침 테스트
- [ ] 업로드 진행 중 페이지 새로고침
- [ ] **확인**: 부분적으로 업로드된 사진 처리 확인

#### 10.3 동일 사진 중복 업로드
- [ ] 동일한 사진 파일을 2번 업로드
- [ ] **예상 결과**: 모두 성공 (파일명이 다르므로 구분됨)
- [ ] **확인**: 2장 모두 그리드에 표시

### ✅ 11단계: 로그 확인

#### 11.1 콘솔 로그 검증
- [ ] 브라우저 개발자 도구 → Console 탭 열기
- [ ] 사진 업로드 시작
- [ ] **확인 항목**:
  - [ ] "📤 업로드 시작:" 로그
  - [ ] "📁 파일 1 업로드 시작:" (각 파일마다)
  - [ ] "✅ 스토리지 업로드 성공:"
  - [ ] "🔗 공개 URL 획득:"
  - [ ] "💾 DB 저장 시도:"
  - [ ] "✅ DB 저장 성공:"
  - [ ] "📊 업로드 완료:"

#### 11.2 에러 로그 검증
- [ ] 크기 초과 파일 업로드 시도
- [ ] **확인**: "❌ 파일 업로드 실패:" 로그
- [ ] **확인**: 에러 상세 정보 표시
